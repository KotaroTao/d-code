# 本格運用コース（Google Cloud）カリキュラム詳細

## コース概要

| 項目 | 内容 |
|------|------|
| コース名 | D-Code Academy 本格運用コース |
| サブタイトル | 「作れた」から「運用できる」へ — Google Cloudで本格展開 |
| 対象 | 導入コース修了者、またはVercel + Supabaseでのアプリ開発経験者 |
| 前提条件 | 導入コース修了（推奨）、Claude Codeの基本操作ができること |
| ゴール | Google Cloud上で本番運用レベルのアプリを構築・運用できる |
| 期間 | 毎週日曜 10:00-12:50 × 5日間（全15時間） |
| 形式 | Zoomオンライン |
| 定員 | 1コホート 15〜20名（少人数制） |
| 技術スタック | Claude Code WEB版 + Next.js + Cloud Run + Cloud SQL + Firebase Auth + Cloud Storage |

### 1日の基本構成（3時間）

| 時間 | ブロック | 内容 |
|------|----------|------|
| 10:00-10:50 | 講義（50分） | GCPサービスの解説と設計思想 |
| 10:50-11:00 | 休憩（10分） | |
| 11:00-11:50 | ハンズオン（50分） | GCP上での実践開発 |
| 11:50-12:00 | 休憩（10分） | |
| 12:00-12:50 | 発表・振り返り（50分） | 成果共有＆ディスカッション |

### コース設計の方針

- **Vercelとの対比で理解**: 「Vercelではこうだった → GCPではこうする」で理解を深める
- **実運用を意識**: セキュリティ、バックアップ、モニタリングなど運用面もカバー
- **Claude Code活用**: GCPの設定もClaude Codeに相談しながら進める
- **歯科医院の実務**: 患者データの取り扱い、HIPAA的な考え方（日本版）

### 導入コースとの比較

| 観点 | 導入コース（Vercel） | 本格運用コース（Google Cloud） |
|------|---------------------|-------------------------------|
| 難易度 | 初心者向け | 中級者向け |
| デプロイ | Vercel（GitHub連携で自動） | Cloud Run（コンテナベース） |
| DB | Supabase（マネージド） | Cloud SQL（PostgreSQL直接管理） |
| 認証 | Supabase Auth | Firebase Authentication |
| ストレージ | Supabase Storage | Cloud Storage |
| 監視 | なし（不要） | Cloud Monitoring + Cloud Logging |
| CI/CD | GitHub → Vercel自動 | Cloud Build or GitHub Actions → Cloud Run |
| 費用 | 無料〜$25/月 | $30〜100/月（無料枠あり） |
| 適性 | プロトタイプ、小規模運用 | 本格運用、スケーラブル |

## Day 1: Google Cloudの世界へ — クラウドインフラの基礎

### テーマ

|------|------|------|

| 10:10-10:25 | クラウドとは何か | 例え：「自分のパソコンの中にサーバーを持つ」→「世界中のデータセンターを借りる」。オンプレミス vs クラウド。AWS/Azure/GCPの位置づけ。なぜGCPか：Googleの信頼性、Firebase連携、無料枠の充実 |

## 導入コースからの移行パス

| 導入コースの要素 | 本格運用コースでの対応 |
|-----------------|---------------------|
| Vercel | → Cloud Run |
| Supabase (DB) | → Cloud SQL (PostgreSQL) |
| Supabase Auth | → Firebase Authentication |
| Supabase Storage | → Cloud Storage |
| Supabase Realtime | → Firestore or Pub/Sub |
| GitHub → Vercel自動デプロイ | → GitHub Actions → Cloud Run |
| Vercel環境変数 | → Secret Manager |
