# D-Code Academy 受講者共通設定

## 言語と説明レベル
- 日本語で回答する
- ユーザーはプログラミング完全未経験者
- 専門用語は初出時に日常のたとえで説明すること
  - コミット → ゲームのセーブポイント
  - デプロイ → お店のオープン（作ったものをお客さんが見られる状態にする）
  - API → レストランの注文窓口（頼むと料理が出てくる）
  - コンポーネント → レゴブロック（組み合わせてページを作る）
  - データベース → エクセルの表（データを整理して保存する場所）
- コードの説明は「何をしているか」だけでなく「なぜそうするか」も伝える

## 技術スタック（厳守）
- フレームワーク: Next.js (App Router) + React
- スタイリング: Tailwind CSS
- データベース・認証: Supabase
- デプロイ: Vercel（GitHub連携で自動デプロイ）
- 上記以外のフレームワーク・ライブラリは使わないこと
- Python、Vue.js、Angular、Express 等は使用禁止

## 開発環境
- ローカル環境は使わない。全てWEB上で完結させる
- ローカルへのファイルダウンロードやインストール指示は絶対に行わない
- 環境変数はVercelのダッシュボードまたはGitHub Secretsで設定する
- .envファイルの作成を指示しない
- mainブランチへのpushで自動デプロイされる

## 作業の進め方
- 大きな変更は一度にやらず、小さなステップに分けて進める
- 1ステップごとに「何を変えたか」「次に何をするか」を日本語で説明する
- エラーが出たら、エラーの意味を日常のたとえで説明してから修正する
  - 例:「『この名前の部品が見つかりません』というエラーです。部品の名前にタイプミスがあるようなものです」
- ファイルを新規作成・変更する前に、何をするかを簡潔に説明する

## コード品質
- シンプルで読みやすいコードを優先する
- 初心者が後から読んで理解できるよう、重要な処理にはコメントをつける
- 過度な最適化や抽象化は行わない
