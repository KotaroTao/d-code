# D-Code Academy サポート講師 募集要項

## 募集ポジション

**D-Code Academy サポート講師（業務委託）**

歯科医院の院長・スタッフ向けバイブコーディング講座で、受講者の学習をサポートする役割です。

---

## 業務内容

| 業務 | 時間 | 内容 |
|------|------|------|
| セミナーサポート | 3h × 5日（15h） | Zoom参加、ブレイクアウトルームで受講者の個別サポート |
| セミナー後の個別相談 | 20h | 受講者からの質問対応（非同期チャット or 30分Zoom） |
| その他 | 10h | 事前準備、講師ミーティング、カリキュラム確認 |
| **合計** | **45h / 1コホート** | |

### 具体的なサポート内容
- 受講者がClaude Codeでエラーが出た際の原因特定・解決サポート
- 「次に何をすればいいかわからない」受講者への声かけ・ガイド
- ブレイクアウトルームでの少人数サポート（5〜6名を担当）
- セミナー後のチャット質問への回答（24h以内目安）
- 必要に応じて30分のZoom個別相談

## 求める人材

### 必須条件
- **Claude Code を日常的に使っている**（最重要）
- Next.js / React の基本的な理解がある
- 日本語でのコミュニケーションに問題がない
- 日曜 10:00-13:00 に5週連続で参加できる

### 歓迎条件
- 教える・サポートした経験（塾講師、メンター、SE研修など）
- Supabase / Vercel の使用経験
- 歯科業界の知識（なくてもOK）

### 不要なもの
- 高度なプログラミングスキル（受講者は完全初心者、バイブコーディングが中心）
- 講師経験（サポート役なので、教え上手であれば十分）

## 応募フォーム項目一覧

応募は `/recruit/support` ページのWebフォーム（またはGoogleフォーム）で受付。
以下のヒアリング項目を設定：

### 基本情報
1. **氏名** *必須
2. **メールアドレス** *必須
3. **SNSアカウント（X / LinkedIn等）** 任意

### Claude Code 関連（最重要カテゴリ）
4. **Claude Code の使用頻度** *必須（選択式）
   - 毎日使っている
   - 週に数回使っている
   - 月に数回使っている
   - 使ったことがある程度
5. **Claude Code の主な利用形態** *必須（複数選択可）
   - Claude Code WEB版（ブラウザ上）
   - Claude Code CLI（ターミナル）
   - VS Code + Claude Code拡張
   - その他のIDE連携
6. **Claude Code で作ったもの・取り組んでいること** *必須（自由記述・200字程度）

### 技術スキル
7. **Next.js / React の経験** *必須（選択式）
   - 実務で使用中
   - 個人開発で使用
   - 学習中
   - ほぼ未経験（Claude Codeで使っている程度）
8. **Vercel の使用経験** *必須（選択式）
   - 使い慣れている
   - 使ったことがある
   - 知っているが未使用
   - 知らない
9. **Supabase の使用経験** *必須（選択式）
   - 使い慣れている
   - 使ったことがある
   - 知っているが未使用
   - 知らない

### サポート適性
10. **教える・サポートの経験** 任意（自由記述・200字程度）
11. **初心者をサポートする際に大切にしていること** *必須（自由記述・200字程度）

### スケジュール・意欲
12. **日曜 10:00-13:00 × 5週連続の参加可否** *必須（選択式）
    - 毎回参加できる
    - ほぼ参加可能（1回程度は調整が必要かも）
    - 半分程度なら参加できる
    - まだ未定だが興味がある
13. **志望動機・ひとこと** *必須（自由記述・200字程度）
14. **ポートフォリオ / GitHub URL** 任意

### 項目設計の意図

| 項目 | 意図 |
|------|------|
| Claude Code使用頻度・利用形態 | 最重要スクリーニング。WEB版経験者は特に優遇（講座で使用するのがWEB版のため） |
| Claude Codeで作ったもの | 実際のアウトプットから習熟度を判断 |
| Vercel / Supabase個別質問 | 講座の技術スタックに沿ったスキルマッピング |
| 初心者サポートで大切にしていること | 技術力ではなく「伝える力」「寄り添う力」を見るための質問 |
| 参加可否の選択肢の細分化 | 「未定だが興味がある」層も逃さない設計 |
