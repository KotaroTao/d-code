# 導入コース（Vercel）カリキュラム詳細

## コース概要

| 項目 | 内容 |
|------|------|
| コース名 | D-Code Academy 導入コース |
| サブタイトル | 「AIでアプリが作れた！」を体験する5日間 |
| 対象 | プログラミング完全未経験者（歯科医院 院長・スタッフ） |
| ゴール | Claude Codeを使って自院オリジナルアプリを開発・公開する |
| 期間 | 毎週日曜 10:00-12:50 × 5日間（全15時間） |
| 形式 | Zoomオンライン |
| 定員 | 1コホート 20〜30名 |
| 技術スタック | Claude Code WEB版 + Next.js + Tailwind CSS + Supabase + Vercel |

### 1日の基本構成（3時間）

| 時間 | ブロック | 内容 |
|------|----------|------|
| 10:00-10:50 | 講義（50分） | その日のテーマを解説 |
| 10:50-11:00 | 休憩（10分） | |
| 11:00-11:50 | ハンズオン（50分） | 手を動かして開発 |
| 11:50-12:00 | 休憩（10分） | |
| 12:00-12:50 | 発表・振り返り（50分） | 成果共有＆フィードバック |

### コース設計の方針

- **感動ファースト**: 毎日必ず「動くもの」をデプロイして達成感を得る
- **ローカルファイルゼロ**: 全ての操作をブラウザ上で完結
- **段階的成長**: 前日の成果に1機能ずつ積み上げる
- **歯科実務直結**: Day 3以降は歯科医院の課題を題材にする

## Day 2: Webアプリの基本 — ToDoアプリでプロンプトを磨こう

### テーマ
「プロンプト（指示の出し方）で成果物の品質が変わる」ことを体感する

### 講義（10:00-10:50）：Web技術とプロンプト設計

| 時間 | 内容 | 詳細 |
|------|------|------|
| 10:00-10:10 | 前回の振り返り | 課題の共有（数名がデプロイURLを見せる）、つまずきポイントの解消 |
| 10:10-10:20 | Webアプリの仕組み | HTML=骨組み、CSS=デザイン、JavaScript=動き。例え：家に例えると、HTML=構造（柱・壁）、CSS=内装（壁紙・家具）、JavaScript=家電（動くもの）。深入りしない |
| 10:20-10:30 | プロンプトエンジニアリング基礎 | 良いプロンプトの3原則：①具体的に（「かっこよくして」→「青を基調に角丸ボタンで」）②段階的に（一度に全部頼まない）③コンテキストを与える（何のためのアプリか伝える） |
| 10:30-10:50 | CLAUDE.mdの活用法 | プロジェクトの「設計書」をClaude Codeに渡す方法。毎回同じ説明をしなくて済む。受講者用テンプレートを一緒に設定 |

### ハンズオン（11:00-11:50）：ToDoアプリ開発

| 時間 | 内容 | 詳細 |
|------|------|------|
| 11:00-11:10 | Next.jsとReactの概要 | なぜ使うのか→Vercelとの相性、世界で最も使われているフレームワークの1つ。コンポーネント思考＝「パーツを組み合わせてページを作る」 |
| 11:10-11:35 | ToDoアプリ作成 | Claude Codeに段階的に指示：①「タスクを追加・表示できるToDoアプリを作って」②「完了チェック機能を追加して」③「削除機能を追加して」。プロンプトの改善を実践 |
| 11:35-11:50 | デザイン調整 | 「背景を暗くして、カードUIにして」「ボタンにホバーエフェクトをつけて」など、見た目の指示を体験 |

### 発表・振り返り（12:00-12:50）：相互レビュー

| 時間 | 内容 | 詳細 |
|------|------|------|
| 12:00-12:20 | オリジナル機能追加 | 各自のToDoアプリに1つ機能追加（例：カテゴリ分け、期限設定、優先度）。プロンプトの書き方を工夫する |
| 12:20-12:40 | デプロイ＆相互レビュー | Vercelにデプロイ。ブレイクアウトルーム（4〜5人）で互いのToDoアプリを体験し合う。「このUI良いね」「この機能面白い」とフィードバック |
| 12:40-12:50 | 振り返り＆課題説明 | プロンプトの改善前後の比較。課題：ToDoアプリにさらに1機能追加 |

### Day 2 到達目標
- [x] プロンプトの書き方で成果物の質が変わることを理解する
- [x] CLAUDE.mdを設定し、プロジェクトの前提を共有する方法を習得する
- [x] 基本的なCRUD操作（作成・読取・更新・削除）を持つアプリを作成できる
- [x] 他の受講者の作品を見て刺激を受ける

### Day 2 課題
- ToDoアプリにオリジナル機能を1つ追加してデプロイ

## Day 3: データベース連携 — 歯科向けアプリを作ろう

### テーマ
「データを保存する」ことで、実用的なアプリに変わる

### 講義（10:00-10:50）：データベースの世界

| 時間 | 内容 | 詳細 |
|------|------|------|
| 10:00-10:10 | 前回の振り返り | 課題の共有、プロンプトのコツ復習 |
| 10:10-10:25 | データベースとは | 例え：「エクセルの表がインターネット上にある」イメージ。行=データ1件、列=項目。ToDoアプリとの違い：ブラウザを閉じてもデータが消えない |
| 10:25-10:35 | Supabaseの紹介 | なぜSupabase？→無料で始められる、ダッシュボードが見やすい、認証機能つき。PostgreSQL=データベースの種類（超有名・超安定） |
| 10:35-10:50 | APIの概念 | 例え：「レストランの注文窓口」。アプリ（お客さん）→API（注文窓口）→データベース（厨房）。注文を出す=データを取得、注文票を書く=データを保存 |

### ハンズオン（11:00-11:50）：歯科向けアプリ開発

| 時間 | 内容 | 詳細 |
|------|------|------|
| 11:00-11:15 | Supabaseセットアップ | アカウント作成、プロジェクト作成、テーブル設計。例：患者お知らせ管理テーブル（タイトル、本文、日付、カテゴリ） |
| 11:15-11:45 | 歯科アプリ開発 | Claude Codeでお知らせ管理アプリを作成。Supabase連携の指示方法。CRUD操作を歯科の文脈で実装（お知らせの追加・一覧・編集・削除） |
| 11:45-11:50 | 環境変数の設定 | Vercelダッシュボードで環境変数を設定。「機密情報をコードに書かない」ことの重要性 |

### 発表・振り返り（12:00-12:50）：実用アプリの可能性

| 時間 | 内容 | 詳細 |
|------|------|------|
| 12:00-12:15 | デプロイ＆動作確認 | Vercelにデプロイ。実際にデータを追加・編集して「保存される」ことを確認 |
| 12:15-12:30 | Supabase停止防止設定 | GitHub Actionsで自動ping設定。ブラウザUIだけで完結する手順を全員で実施 |
| 12:30-12:45 | 歯科アプリアイデア共有 | ブレイクアウトルームで「自院でこんなアプリがあったら便利」を3つずつ出す。予約管理、在庫管理、スタッフシフト、患者アンケート等 |
| 12:45-12:50 | 振り返り＆課題説明 | Day 4でのオリジナルアプリ開発に向けた準備。課題：歯科アプリにログイン機能を追加（Supabase Auth） |

### Day 3 到達目標

- [x] 環境変数の設定方法を理解する
- [x] GitHub Actionsでの自動化を体験する

### Day 3 課題
- 歯科アプリにログイン機能（Supabase Auth）を追加してデプロイ

## 本格運用コースへの接続

導入コース修了者には、本格運用コース（Google Cloud）へのステップアップを案内：
- 導入コースで作ったアプリをGCPに移行
- エンタープライズレベルの運用スキルを習得
- 本格的なスケーラビリティ・セキュリティを実現
