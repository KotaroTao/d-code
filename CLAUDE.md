# D-Code Academy — プロジェクト方針

## 最重要原則：ローカルファイルゼロ

本講座は **ローカルに一切ファイルを置かない** ことが大前提。
全ての開発・設定・デプロイをWEB上で完結させる。

- コード編集: Claude Code WEB版（ブラウザ上）
- バージョン管理: GitHub（ブラウザUI）
- デプロイ: Vercel（GitHub連携で自動）
- データベース: Supabase（ブラウザダッシュボード）
- CI設定: GitHub Actions（ブラウザUIで設定）

**「ターミナルを開く」「ローカルにcloneする」「npm installする」といった指示は絶対に行わない。**
受講者のPCにはブラウザ以外のツールを一切インストールさせない。

## 対象者の前提

- プログラミング完全未経験者
- 「HTML」「API」「GitHub」などの用語を知らない前提で説明する
- 専門用語を使う場合は必ず日用品や日常の例えで補足する
- 「コードを書く」ではなく「Claude Codeに指示を出す」という表現を基本とする

## 講座設計の原則

### 1. 感動ファースト
- Day 1で「自分のサイトがインターネットに公開される」体験を最優先する
- 理論の説明は最小限にし、まず動くものを見せてから仕組みを解説する
- 毎日必ずデプロイして「動いている」実感を持たせる

### 2. 段階的な難易度設計
- Day 1: 静的ページ — 自己紹介サイト（HTML/CSS）
- Day 2: インタラクティブアプリ — ToDoアプリ（React/Next.js）+ プロンプト設計
- Day 3: データ永続化 — 歯科向けアプリ（Supabase連携）
- Day 4: オリジナルアプリ開発 — 自院の課題を解決するアプリ
- Day 5: 仕上げ・発表 — UI改善、独自ドメイン、成果発表会
- 各Dayの課題は「前日の内容に1機能追加」程度の負荷にする
- **Day 3以降は歯科医院の実務に即した題材を使う**（汎用的なメモアプリ等ではなく）

### 3. プロンプト品質の指導
- 受講者が書くプロンプトの質が成果物の質に直結する
- Day 2でプロンプトエンジニアリングを重点的に扱う
- CLAUDE.mdの活用方法を教え、プロジェクトごとの指示書を作る習慣をつけさせる

## 技術スタック（固定）

| 用途 | ツール | 選定理由 |
|------|--------|----------|
| AI開発 | Claude Code WEB版 | ローカル不要、ブラウザ完結 |
| ソース管理 | GitHub | Vercel連携、Actions対応 |
| フレームワーク | Next.js (React) | Vercelとの親和性、エコシステムの豊富さ |
| デプロイ | Vercel | GitHub push で自動デプロイ、無料枠あり |
| DB/認証 | Supabase | 無料枠が十分、リアルタイム対応、ダッシュボードUI |
| CSS | Tailwind CSS | Claude Codeとの相性が良い、クラスベースで直感的 |

技術スタックは講座全体で統一する。受講者ごとに異なるツールを使わせない。

## 運営上の注意点

### Claude Pro プランの使用量制限
- 受講者は全員Claude Pro（$20/月）が必要
- 3時間の集中利用でレートリミットに当たる可能性がある（唯一の実質的リスク）
- 対策：プロンプトを雑に乱打しない習慣づけ、CLAUDE.mdで前提を設定し毎回の説明コストを省く、大きな変更は分割して依頼
- リミット到達時は設計書を書く・他の受講者の作品を見る等で時間を有効活用
- 深刻な場合は Max $100/月プランを案内

### Supabase Freeプランの停止
- 7日間非アクティブで自動停止される
- Day 3でGitHub Actionsによる自動ping設定を全員で行う（ブラウザUIで完結）
- 停止しても90日以内ならダッシュボードからワンクリック復元可能

### Vercel プランについて
- Academy運営側のサイトは Vercel Pro（$20/月）が必須（商用利用のため）
- 受講者は Hobby（無料）で問題なし（学習用途）
- 講座後に商用展開する場合はPro への移行を案内

## デプロイフロー（受講者が行う操作）

```
Claude Code WEB版でコード編集
    ↓ （自動でGitHubにcommit/push）
GitHub リポジトリに反映
    ↓ （Vercelが自動検知）
Vercel で自動ビルド＆デプロイ
    ↓
公開URL でアプリが更新される
```

受講者が意識するのは「Claude Codeに指示を出す」ことだけ。
デプロイは自動で行われる仕組みをDay 1で構築する。

## D-Code プロジェクト全体における位置づけ

D-Code Academyは、D-Codeプロジェクトの3本柱の1つ：

| 柱 | 内容 | 収益モデル |
|----|------|------------|
| **Academy** | Claude Code × 歯科のバイブコーディング講座 | 受講料 ¥350,000 × 人数 |
| **Store** | 歯科専用アプリマーケットプレイス | 販売手数料（卒業生が出品可能） |
| **Hub** | レセコン統一API基盤 | API利用料（月額課金） |

Academy → Club → Store → Hub の流れで、受講者を長期的なエコシステムに取り込む。
Y3目標：年商 ¥10億、利益 ¥5億以上。
